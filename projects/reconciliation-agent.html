Reconciliation-Report Agent

An AI agent that reconciles records across systems and writes the report staff used to assemble by hand.

  • AI Agents
  • Layered Matching
  • Human-in-the-loop
  • Reporting
  • Back-Office

The Problem

Reconciliation — the work of matching and cross-checking records across systems — was a repetitive, time-consuming, and error-prone task done manually by the team every cycle.

What I Built

A layered matching engine: rules clear the obvious matches, fuzzy matching handles near-matches, and an LLM is used only for the genuinely ambiguous pairs. Only true discrepancies reach a human for a one-click confirm/dismiss — and every decision refines the rules. An LLM then drafts the finished reconciliation summary, delivered on a schedule.

How It Works

1 · RULESRules clear the obvious matches instantly.
2 · FUZZYFuzzy matching handles the near-matches.
3 · LLMLLM resolves only the genuinely ambiguous pairs.
4 · HUMANTrue discrepancies get a one-click confirm/dismiss.
5 · REPORTLLM drafts & delivers the report on schedule.

Why It Matters

  • Removes a repetitive, manual admin task.
  • Reduces human error in matching and cross-checking.
  • Frees staff for higher-value work.

Tech Stack

  • Rule + fuzzy + LLM matching
  • Human-in-the-loop
  • LLM reporting
  • Python / pandas
  • Scheduled jobs
