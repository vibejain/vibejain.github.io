Ops Intelligence Monitor

Event-driven agent that tracks live operational state, detects SLA breaches and anomalies, and fires smart escalations before issues surface as complaints.

  • Python
  • State Machine
  • LLM
  • Webhooks
  • Escalation

The Problem

Keeping stakeholders informed on fast-moving operational events meant someone manually checking status and writing updates. It didn't scale, communications were often late or missed, and worse — issues surfaced only when a stakeholder complained, by which point the window to act had already closed.

What I Built

An event-driven monitoring agent that polls live data sources, tracks state changes for every active item, and triggers the right communication automatically. On a routine status transition it sends a timely, personalised update. When something breaches its SLA or hits an exception, it fires a proactive alert to the stakeholder and escalates to the internal team — giving them time to act before it becomes a complaint. Each event is communicated exactly once; no duplicate noise.

How It Works

1 · POLLPoll live data sources for every active item.
2 · DIFFCompare each item against its last known state.
3 · NOTIFYSend personalised status updates — exactly once per event.
4 · ESCALATEOn SLA breach or exception, alert stakeholder + escalate internally.

Why It Matters

  • Stakeholders stay informed at every milestone without manual effort.
  • SLA breaches become early, actionable warnings instead of silent failures.
  • Internal teams are escalated in time to act, protecting relationships.
  • Eliminates a recurring, low-value manual task and the cost of late surprises.

Tech Stack

  • Python polling loop
  • State machine
  • SLA / ETA detection
  • LLM-generated messages
  • Webhook / email delivery
  • Escalation routing
