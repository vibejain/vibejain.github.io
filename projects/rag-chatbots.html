RAG-Enabled Chatbots: Self-Hosted & Cloud LLMs

Chatbots that answer from a company's own documents and data — grounded, accurate, and deployed end to end.

  • RAG
  • Llama on vLLM

The Problem

Off-the-shelf LLMs don't know a company's internal knowledge, and sending sensitive data to a third party isn't always acceptable. Teams needed a chatbot that answered from their documents — reliably, without leaking data or hallucinating.

What I Built

A retrieval-augmented generation (RAG) system with a deliberate dual-model strategy:

Self-hosted Llama (served on vLLM, RunPod GPU pods) for data-sensitive, cost-controlled workloads — so private data never has to leave the environment. Anthropic Claude API for tasks that needed stronger reasoning and higher-quality generation. Responses are tied back to source context to keep answers trustworthy and reduce hallucination.

How It Works

1 · INGESTIngest & chunk source documents.
2 · EMBEDGenerate embeddings, store them in Qdrant vector DB.
3 · RETRIEVEOn each query, retrieve the most relevant context.
4 · GROUNDPass context to Llama or Claude for a grounded answer.

Why It Matters

  • Keeps sensitive data in-house via self-hosting where required.
  • Answers are grounded in real documents, not generic model memory.
  • Flexible model routing balances cost, privacy, and reasoning quality.
